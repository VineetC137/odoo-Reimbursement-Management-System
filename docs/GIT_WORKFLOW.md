# Git Workflow

## Branching Model
- Trunk-based with short-lived feature branches:
  - `feat/*`
  - `fix/*`
  - `docs/*`

## Commit Convention
- `feat:`
- `fix:`
- `refactor:`
- `docs:`
- `test:`
- `chore:`

## Cadence
- Small, meaningful commits every 2–4 hours.
- PRs under ~500 lines when possible.

## Pull Request Template Requirements
- Summary
- Screenshots (if UI changed)
- Test notes
- Migration notes
- Reviewer checklist

## Ownership
- Every contributor must have visible commits in history.
- Avoid single-author bulk drops near deadlines.
